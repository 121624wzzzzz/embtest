## 实验1：全局几何相似性验证 (exp1_global_geometry)

### 目标

计算不同模型嵌入空间的**全局几何相似性**：通过随机采样 token 对，计算两个模型的 **Embedding (E)** 和 **Unembedding (U)** 矩阵的**距离矩阵相关系数 (GCorr)**。

### 核心参数

| 参数 | 取值 | 说明 |
|------|------|------|
| `n_tokens` | 20,000（上限） | 每次对比实际采样 token 数为 \(n\\_sample=\\min(20000,\\text{可用token数})\)，并写入输出 CSV 的 `n_tokens` 字段 |
| `n_pairs` | 5,000,000 | 用于计算距离矩阵的 token 对数量 |
| `n_bootstrap` | 10 或 100 | Bootstrap 重采样次数；以合并后的 `results/exp1_global_v4_complete.csv` 为准：Base–Instruct 的 19 对使用 10，其余对比使用 100 |
| `random_seed` | 42 | 随机种子 |

### 一次对比“怎么做出来的”（从矩阵到一行 CSV）

对每一组模型对比 (A,B)，脚本会输出一行结果，流程是：

1. **抽取矩阵**：读取两模型的输入嵌入 \(E\\) 与输出端矩阵 \(U\\)（LM head / unembedding）。若模型无 `lm_head` 则用 `U=E`。同时记录 `is_tied`（config 声明）与 `actual_tied`（`E` 与 `U` 的 allclose 实测）。
2. **构造可用 token 集**：从 tokenizer 的 `0..vocab_size-1` 中剔除 `bos/eos/pad/unk` 与 `all_special_ids`。
3. **决定 token 对齐方式**：
   - **同 tokenizer（脚本的操作性定义：`vocab_size_a == vocab_size_b`）**：直接在同一组 token id 上采样，并对同批次中的所有模型对共享同一套 `sample_ids`。
   - **跨 tokenizer**：按 token **字符串**对齐得到公共 token（`convert_ids_to_tokens` 建 map 后取交集），再在公共 token 上采样；若公共 token < 1000 则跳过该对比。
4. **Bootstrap**：重复 `n_bootstrap` 次。第 \(b\\) 次 bootstrap 使用 `seed = random_seed + b`，该 seed 同时用于 token 采样与距离对采样。
5. **距离矩阵抽样**：在采样到的 \(n\\_sample\\) 个 token 上，从所有 \((i<j)\) 对中抽样 `n_pairs=5,000,000` 个 token 对（GPU 上拒绝采样生成索引），避免显式构造完整距离矩阵。
6. **计算 GCorr**：
   - 对 **E** 与 **U** 分别计算三类距离/相似度：cos、euc、euc2；
   - 对每一类，把模型 A 的距离向量与模型 B 的距离向量做 Pearson 相关，得到该 bootstrap 的 GCorr。
7. **汇总**：对每个指标在 bootstrap 维度上输出 `mean/std/se/ci95_low/ci95_high` 等统计量，写入 CSV。

### 实验分组与规模（以 v4 输出为准）

实验1的所有对比在分析时使用同一份合并后的结果文件：`results/exp1_global_v4_complete.csv`。该文件共 **297 组模型对比**（每行一组模型对比），其中：

- **同系列对比**：189 组（含 Base–Instruct、以及系列内部的完整两两对比；其中自然包含不同规模之间的小↔大组合）
- **跨系列对比**：108 组（按 1.3 的同规模分组与 1.4 的 MoE 相关对比组织）

另外，`exp1_global_v4_complete.csv` 中仍保留了每行的运行参数字段（如 `n_tokens/n_bootstrap/n_pairs`），其中 Base–Instruct 的 19 对为 `n_bootstrap=10`，其余大部分对比为 `n_bootstrap=100`；跨 tokenizer 情况下 `n_tokens` 可能因公共 token 数不足而小于 20000（写入该行的 `n_tokens` 字段）。

## 实验2：同系列内跨模型 E–E / U–U 仿射关系检验 (exp2_affine_cross_model)

### 目标

对**同系列**内的两个模型 A、B，检验：

- **E 与 E**：A 的 embedding 与 B 的 embedding 是否满足仿射关系（E_B ≈ A @ E_A + b，在公共 token 上）。
- **U 与 U**：A 的 unembedding 与 B 的 unembedding 是否满足仿射关系（U_B ≈ A @ U_A + b）。

即**跨模型的 E–E、U–U 仿射**，以及**同一模型内部的 E–U 仿射**。

### 核心参数

| 参数 | 取值 | 说明 |
|------|------|------|
| `min_common_tokens` | 5,000 | 公共 token 数至少达到此值才做仿射拟合 |
| `max_fit_rows` | 24,000 | 拟合时最多使用的行数（控制显存）；若公共 token 数超过此值则随机抽样到此数 |

### 结果文件所对应的“实际规模与采样”（来自 `results/*.csv`）

- **跨模型（同系列内两两）**：`results/affine_cross_model_results.csv` 共 **189 对**，按系列分别为：Qwen3=55、Qwen2.5=91、Llama=28、Gemma2=15（全部跑满两两组合）。
- **模型内（每模型一次 E→U）**：`results/affine_intra_model_EU_results.csv` 共 **39 个模型**（Qwen3=11、Qwen2.5=14、Llama=8、Gemma2=6）。
- **拟合行数**：在当前输出里，所有跨模型对比的 `n_fit` **恒为 24000**（即全部都触发了 `max_fit_rows` 上限；公共 token 数 `n_common` 均远大于 24k）。

### 矩阵加载、token 对齐与拟合流程（实现口径）

1. **矩阵读取**：从模型目录的 `*.safetensors` 读取 \(E\\) 与 \(U\\)。`E` 关键名优先 `model.embed_tokens.weight`/`transformer.wte.weight`；`U` 优先 `lm_head.weight`/`output.weight`；若找不到 `U` 则令 `U=E`。
2. **公共 token 对齐**：先剔除 special token id，再按 token 字符串交集对齐得到 `(ids_a, ids_b)`（一一对应同一 token 字符串）。
3. **行数控制与复现性**：若公共 token 数超过 24k，则用确定性种子对行索引无放回抽样到 24k（种子由模型名字符 `ord` 求和构造，避免 Python hash 随机化导致不可复现）。
4. **拟合方式**：采用“中心化后最小二乘”的一般仿射拟合（先拟合线性项 \(A\\)，再恢复平移项 \(b\\)），并分别对 E–E、U–U（以及模型内 E→U）输出 R²、相对误差、\|A\|_F、\|b\|_2 等字段。

### 模型与组别

- **4 个系列**：Qwen3、Qwen2.5、Llama、Gemma2。
- **系列内模型**（来自 `run_affine_cross_model.py` 的 FAMILIES）：
  - **Qwen3**（11）：Qwen3-0.6B-Base, Qwen3-0.6B, Qwen3-1.7B-Base, Qwen3-1.7B, Qwen3-4B-Base, Qwen3-4B, Qwen3-8B-Base, Qwen3-8B, Qwen3-14B-Base, Qwen3-14B, Qwen3-32B
  - **Qwen2.5**（14）：Qwen2.5-0.5B, Qwen2.5-0.5B-Instruct, Qwen2.5-1.5B, Qwen2.5-1.5B-Instruct, Qwen2.5-3B, Qwen2.5-3B-Instruct, Qwen2.5-7B, Qwen2.5-7B-Instruct, Qwen2.5-14B, Qwen2.5-14B-Instruct, Qwen2.5-32B, Qwen2.5-32B-Instruct, Qwen2.5-72B-Base, Qwen2.5-72B-Instruct
  - **Llama**（8）：Llama-3.2-1B, Llama-3.2-1B-Instruct, Llama-3.2-3B, Llama-3.2-3B-Instruct, Llama-3.1-8B, Llama-3.1-8B-Instruct, Llama-3.1-70B-Base, Llama-3.1-70B-Instruct
  - **Gemma2**（6）：Gemma-2-2B, Gemma-2-2B-Instruct, Gemma-2-9B, Gemma-2-9B-Instruct, Gemma-2-27B, Gemma-2-27B-Instruct

### 实验规模（模型对数量）

- **跨模型 E–E / U–U**：每个系列内所有模型**两两组合**。
  - Qwen3：C(11,2) = 55 对  
  - Qwen2.5：C(14,2) = 91 对  
  - Llama：C(8,2) = 28 对  
  - Gemma2：C(6,2) = 15 对  
  - **合计**：189 对（若公共 token 数 ≥ min_common_tokens 则均有结果）。
- **模型内部 E–U**：每个系列内每个模型单独做一次 E→U 仿射拟合，共 11+14+8+6 = **39 个模型**。

### 方法简述

1. 按 token **字符串对齐**得到两模型的公共 token，得到 E_A_c、E_B_c、U_A_c、U_B_c。
2. 若公共 token 数 > `max_fit_rows`，则用固定种子**随机抽样**到 `max_fit_rows` 行再拟合。
3. 仿射形式：E_B ≈ E_A @ A^T + b（行向量），用**最小二乘**求解 A、b。
4. 对 U 同样拟合 U_B ≈ U_A @ A_U^T + b_U。
5. 报告 **R²、相对误差、||A||_F、||b||_2** 等。

### 指标

- **跨模型**：R2_E、R2_U、rel_err_E、rel_err_U、norm_A_E、norm_b_E、norm_A_U、norm_b_U 等。
- **模型内部 E–U**：R2_EU、rel_err_EU、norm_A_EU、norm_b_EU 等。

### 运行方式

- `python run_affine_cross_model.py`，结果输出在 `results/`。

## 对比小结

| 项目 | 实验1（全局几何） | 实验2（仿射关系） |
|------|------------------|-------------------|
| **主要问题** | 两模型 E/U 距离矩阵是否相关（GCorr） | 同系列两模型 E–E、U–U 是否仿射（及单模型 E–U） |
| **采样方式** | 最多 20k token、500 万对；bootstrap：Base–Instruct=10 次，其余=100 次（以 v4 merged CSV 为准） | 公共 token 字符串对齐；行数上限 24k（当前输出里所有对比 `n_fit=24000`） |
| **组别** | Base–Instruct、系列内两两（排除 Base–Instruct）、跨系列同规模、MoE | 4 系列；系列内两两（E–E/U–U）+ 每模型一次 E→U |
| **规模** | 合并口径总计 297 组（245 + 52） | 跨模型 189 对 + 模型内 39 个 |

以上即为两个实验的**实验设置**总结，不包含结果解读。
